# Cloud

Notes on cloud platforms, services, and cloud-native architecture.

## Topics

- **AWS Services** - Core AWS services and when to use them
- **Azure Basics** - Azure fundamentals and key services

## Cloud Architecture Principles

1. **Design for failure** - Assume any component can fail at any time
2. **Decouple components** - Use queues and events between services
3. **Scale horizontally** - Add more instances, not bigger ones
4. **Automate everything** - Infrastructure as Code, auto-scaling, auto-healing
5. **Pay for what you use** - Right-size resources, use spot/preemptible instances
